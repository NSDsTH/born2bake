**1. Objective**

The objective of this assignment is to demonstrate a systematic approach to software quality assurance. Students must validate their Flutter applications against specific functional requirements (R1-R5) using automated testing and evaluate user experience through performance metrics and usability workshops.

**2. Requirements to be Validated (R1-R5)**

Your test report must provide evidence that the following requirements are met:

•     R1: Authentication: Secure Login/Logout flows using Firebase.

•     R2: Navigation: Correct routing between at least 5 distinct application pages.

•     R3: Cross-Platform Execution: Successful operation on both Android (Emulator/Device) and Web (Chrome).

•     R4: Data Storage: Integration with Cloud Firestore and Local Storage (e.g., Shared Preferences).

•     R5: Testability: The project must be runnable via Command-Line Interface (CLI) for grading verification.

**3. Scope of Work**

**Phase A:** Automated Testing (Unit, Widget, & Integration)

Students must implement a suite of automated tests including:

1.    Unit Tests: Focus on business logic and State Management. Use Mocking to isolate Firebase services.

2.    Widget Tests: Validate UI component rendering and basic user interactions.

3.    Integration Tests: Execute End-to-End (E2E) flows on actual target platforms (Android and Web).

**Phase B: **Performance & Usability Metrics (Weeks 6-7)

Analyze the efficiency of your application using the following metrics:

•     Task Success Rate: Percentage of tasks completed successfully by users.

•     Time-On-Task: Average time taken to complete specific functions.

•     Error Rate: Frequency of user errors or system exceptions during use.

•     Efficiency: Comparison of task completion speed vs. accuracy.

•     Learnability: Improvement in performance when a user repeats a task.

**4. Report Structure**

Your final report must follow the provided Software Testing Report Template and include:

* 1.    Overview & Execution Commands: How to run your tests via CLI.
* 2.    Testing Strategy: Explanation of your "Testing Pyramid" approach.
* 3.    Black-Box Design: Use of Equivalence Partitioning (EP) and Boundary Value Analysis (BVA) for input validation.
* 4.    Requirements Traceability Matrix (RTM): Mapping requirements to specific test scripts.
* 5.    Performance Dashboard: Visualization or table of the metrics from Phase B.
* 6.    Defect Report: Documentation of at least one major bug found and how it was resolved.

5. Grading Criteria (Rubric)

Criteria    Weight      Description
